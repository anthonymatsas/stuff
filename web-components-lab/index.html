1. Hello Element

The simplest custom element: extend HTMLElement, define behavior in connectedCallback (fires when the element is inserted into the DOM), register a tag name with customElements.define. Tag names must contain a hyphen.

Demo
Code
class HelloElement extends HTMLElement {
  connectedCallback() {
    const name = this.getAttribute('name') || 'world';
    this.textContent = `Hello, ${name}!`;
  }
}
customElements.define('hello-element', HelloElement);
<!-- usage -->
<hello-element name="Tony"></hello-element>
<hello-element></hello-element>

2. Shadow DOM

Attach a shadow root and the element gets its own DOM tree. Styles inside stay inside; styles outside don't reach in. The h3 below is bright red inside the component but normal everywhere else on this page.

Demo
This h3 is red, padding is custom — and none of it leaks out.

Same h3 tag outside — unaffected.

Code
class StyledCard extends HTMLElement {
  connectedCallback() {
    const shadow = this.attachShadow({ mode: 'open' });
    shadow.innerHTML = `
      <style>
        .card { padding: 16px; background: #eef2ff;
                border-radius: 8px; }
        h3    { color: tomato; margin: 0 0 8px; }
      </style>
      <div class="card">
        <h3>${this.getAttribute('title')}</h3>
        <slot></slot>
      </div>
    `;
  }
}
customElements.define('styled-card', StyledCard);
Why it matters: No more CSS naming conventions (BEM etc.) to avoid collisions. The browser enforces isolation for you.

3. Reactive Attributes

Declare which attributes you care about with observedAttributes, and the browser calls attributeChangedCallback whenever they change. Click the buttons — the badge updates because we change its count attribute.

Demo
Notifications:
Code
class CounterBadge extends HTMLElement {
  static observedAttributes = ['count'];

  connectedCallback() {
    this.attachShadow({ mode: 'open' });
    this.render();
  }

  attributeChangedCallback(name, oldVal, newVal) {
    if (this.shadowRoot) this.render();
  }

  render() {
    const n = Number(this.getAttribute('count') || 0);
    this.shadowRoot.innerHTML = `
      <style>
        :host { display: inline-block; padding: 2px 10px;
                background: ${n > 0 ? '#ef4444' : '#9ca3af'};
                color: white; border-radius: 999px;
                font-size: 13px; font-weight: 600; }
      </style>
      ${n}
    `;
  }
}
customElements.define('counter-badge', CounterBadge);
Attribute vs property: Attributes are strings in HTML. Properties are JS values on the element instance. For complex data (objects/arrays), use properties; for simple strings, attributes are fine.

4. Slots & Composition

Slots let users pass content into your component. Default slot catches anything; named slots route content to specific spots. The component owns layout, the user owns content.

Demo
Quarterly Report

Revenue is up 12% over Q3, driven by enterprise deals closing earlier than forecast.
Code
class SlottedCard extends HTMLElement {
  connectedCallback() {
    this.attachShadow({ mode: 'open' }).innerHTML = `
      <style>
        article { border: 1px solid #e3e6ee;
                  border-radius: 8px; overflow: hidden; }
        header  { background: #4f46e5; color: white;
                  padding: 10px 14px; font-weight: 600; }
        .body   { padding: 14px; }
        footer  { padding: 10px 14px; background: #f9fafb;
                  display: flex; gap: 8px; }
      </style>
      <article>
        <header><slot name="title">Untitled</slot></header>
        <div class="body"><slot></slot></div>
        <footer><slot name="actions"></slot></footer>
      </article>
    `;
  }
}
customElements.define('slotted-card', SlottedCard);
<!-- usage -->
<slotted-card>
  <span slot="title">Quarterly Report</span>
  <p>Revenue is up 12%...</p>
  <div slot="actions">
    <button>Open</button>
  </div>
</slotted-card>

5. Custom Events

Components communicate up the tree by dispatching events. Use CustomEvent with bubbles: true so ancestors can listen. Click the heart — a parent listener catches the event.

Demo
Code
class LikeButton extends HTMLElement {
  #count = 0;

  connectedCallback() {
    this.attachShadow({ mode: 'open' }).innerHTML = `
      <style>
        button { font-size: 16px; padding: 6px 12px;
                 border: 1px solid #e3e6ee; background: white;
                 border-radius: 6px; cursor: pointer; }
        button:hover { background: #fef2f2; }
      </style>
      <button>❤️ <span>0</span></button>
    `;
    this.shadowRoot.querySelector('button')
      .addEventListener('click', () => this.bump());
  }

  bump() {
    this.#count++;
    this.shadowRoot.querySelector('span').textContent = this.#count;
    this.dispatchEvent(new CustomEvent('like', {
      detail: { count: this.#count },
      bubbles: true,
      composed: true  // cross shadow boundary
    }));
  }
}
customElements.define('like-button', LikeButton);

// listener on a parent
document.getElementById('like-demo').addEventListener('like', e => {
  console.log('liked!', e.detail.count);
});
Watch out: Events from inside a shadow root won't escape unless you set composed: true.

6. Form-Associated Custom Element

A custom element can act like a real form control — submitted with the form, reset with the form, validated with the form — by setting static formAssociated = true and using ElementInternals. This is one of the most powerful (and least known) web component features.

Demo
Code
class ToggleSwitch extends HTMLElement {
  static formAssociated = true;
  #internals; #on = false;

  constructor() {
    super();
    this.#internals = this.attachInternals();
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback() {
    this.#on = this.getAttribute('value') === 'on';
    this.render();
    this.shadowRoot.querySelector('button')
      .addEventListener('click', e => {
        e.preventDefault();
        this.#on = !this.#on;
        this.#sync();
        this.render();
      });
    this.#sync();
  }

  #sync() {
    this.#internals.setFormValue(this.#on ? 'on' : 'off');
  }

  formResetCallback() {
    this.#on = this.getAttribute('value') === 'on';
    this.#sync();
    this.render();
  }

  render() {
    this.shadowRoot.innerHTML = `
      <style>
        button { width: 44px; height: 24px; border: 0;
                 background: ${this.#on ? '#4f46e5' : '#cbd5e1'};
                 border-radius: 999px; cursor: pointer;
                 position: relative; transition: background .15s; }
        button::after { content: ''; position: absolute;
                        top: 2px; left: ${this.#on ? '22px' : '2px'};
                        width: 20px; height: 20px;
                        background: white; border-radius: 50%;
                        transition: left .15s; }
      </style>
      <button type="button" aria-pressed="${this.#on}"></button>
    `;
  }
}
customElements.define('toggle-switch', ToggleSwitch);

7. Full Form Demo

Combining the toggle with a custom <rating-stars> input. Both participate in the form natively — submit and you'll see the values come through standard FormData, no custom serialization needed.

Demo

              

            

          

          

            
Code (RatingStars)


class RatingStars extends HTMLElement {
  static formAssociated = true;
  static observedAttributes = ['value', 'max'];
  #internals;

  constructor() {
    super();
    this.#internals = this.attachInternals();
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback() { this.render(); this.#sync(); }
  attributeChangedCallback() {
    if (this.isConnected) { this.render(); this.#sync(); }
  }
  formResetCallback() { this.render(); this.#sync(); }

  #sync() {
    this.#internals.setFormValue(this.getAttribute('value') || '0');
  }

  render() {
    const max = +(this.getAttribute('max') || 5);
    const val = +(this.getAttribute('value') || 0);
    const stars = Array.from({ length: max }, (_, i) => `
      <button type="button" data-i="${i + 1}">
        ${i < val ? '★' : '☆'}
      </button>
    `).join('');

    this.shadowRoot.innerHTML = `
      <style>
        button { background: none; border: 0; cursor: pointer;
                 font-size: 22px; color: #f59e0b; padding: 2px; }
      </style>${stars}`;

    this.shadowRoot.querySelectorAll('button').forEach(b => {
      b.onclick = () => this.setAttribute('value', b.dataset.i);
    });
  }
}
customElements.define('rating-stars', RatingStars);

// reading the form
form.addEventListener('submit', e => {
  e.preventDefault();
  const data = Object.fromEntries(new FormData(form));
  // => { name: '...', subscribe: 'on', rating: '4' }
});

          

        

      


      
      

        
8. SVG Bar Chart

        

          A reactive chart that re-renders whenever its data attribute changes.
          The component owns layout, axes, and styling; the host page just hands it numbers.
        


        

          

            
Demo

            

              
              

                
                
              

            

          

          

            
Code


class BarChart extends HTMLElement {
  static observedAttributes = ['data'];

  connectedCallback() {
    this.attachShadow({ mode: 'open' });
    this.render();
  }

  attributeChangedCallback() {
    if (this.shadowRoot) this.render();
  }

  render() {
    let data = [];
    try { data = JSON.parse(this.getAttribute('data') || '[]'); }
    catch { data = []; }

    const w = 360, h = 160, pad = 8;
    const max = Math.max(...data, 1);
    const bw = (w - pad * 2) / data.length;

    const bars = data.map((v, i) => {
      const bh = (v / max) * (h - pad * 2);
      return `<rect x="${pad + i * bw + 2}"
        y="${h - pad - bh}" width="${bw - 4}" height="${bh}"
        fill="#4f46e5" rx="3"></rect>
        <text x="${pad + i * bw + bw / 2}" y="${h - pad + 4}"
          text-anchor="middle" font-size="10" fill="#5b6478"
          dy="10">${v}</text>`;
    }).join('');

    this.shadowRoot.innerHTML = `
      <svg viewBox="0 0 ${w} ${h + 14}" width="100%">
        ${bars}
      </svg>
    `;
  }
}
customElements.define('bar-chart', BarChart);

9. Live Sparkline

        

          Internal state + setInterval driven from connectedCallback,
          cleaned up in disconnectedCallback. Hit "Toggle" to remove the element
          from the DOM and watch the cleanup fire (open the console).
        


        

          

            
Demo

            

              

                
              

              

                
              

              

                Open the console to see lifecycle logs.
              

            

          

          

            
Code


class LiveSparkline extends HTMLElement {
  #data = Array.from({ length: 30 }, () => 50);
  #timer;

  connectedCallback() {
    console.log('sparkline mounted');
    this.attachShadow({ mode: 'open' });
    this.render();
    this.#timer = setInterval(() => {
      this.#data.shift();
      this.#data.push(20 + Math.random() * 80);
      this.render();
    }, 500);
  }

  disconnectedCallback() {
    console.log('sparkline unmounted — clearing timer');
    clearInterval(this.#timer);
  }

  render() {
    const w = 360, h = 80;
    const step = w / (this.#data.length - 1);
    const pts = this.#data
      .map((v, i) => `${i * step},${h - (v / 100) * h}`)
      .join(' ');
    const last = this.#data[this.#data.length - 1].toFixed(0);

    this.shadowRoot.innerHTML = `
      <style>
        :host { display: block; }
        .wrap { display: flex; align-items: center; gap: 12px; }
        .val  { font-family: monospace; font-size: 24px;
                color: #4f46e5; min-width: 36px; }
      </style>
      <div class="wrap">
        <div class="val">${last}</div>
        <svg viewBox="0 0 ${w} ${h}" width="100%" height="${h}">
          <polyline points="${pts}" fill="none"
            stroke="#4f46e5" stroke-width="2" />
        </svg>
      </div>
    `;
  }
}
customElements.define('live-sparkline', LiveSparkline);

            
Lifecycle cheat sheet: connectedCallback = inserted, disconnectedCallback = removed, attributeChangedCallback = observed attribute changed, adoptedCallback = moved to a new document.

          

        

      


      
      

        
10. Putting It Together

        

          A complete page — what you'd actually ship. A form with a real <input>
          plus two custom inputs, a custom card rendered per submission, and a live chart.
          One file. No build. The iframe below loads mini-app.html directly from disk.
        


        

          
          

            Open standalone →
          

        


        
Anatomy of the page

        

          Top-to-bottom, here's what every web components page looks like.
        



<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>My App</title>
  <style>
    /* Hide custom elements until they're registered. */
    :not(:defined) { visibility: hidden; }
  </style>
</head>
<body>
  <!-- 1. Use your custom tags like any HTML element. -->
  <form id="form">
    <input name="name" type="text">
    <toggle-switch name="subscribe"></toggle-switch>
    <rating-stars name="rating" value="3"></rating-stars>
    <button type="submit">Submit</button>
  </form>

  <!-- 2. Define them. Order doesn't matter — see below. -->
  <script>
    class ToggleSwitch extends HTMLElement { /* ... */ }
    customElements.define('toggle-switch', ToggleSwitch);

    class RatingStars extends HTMLElement { /* ... */ }
    customElements.define('rating-stars', RatingStars);

    // 3. Page-level wiring like normal JS.
    document.getElementById('form').addEventListener('submit', e => {
      e.preventDefault();
      const data = Object.fromEntries(new FormData(e.target));
      // => { name: '...', subscribe: 'on', rating: '4' }
    });
  </script>
</body>
</html>


        
How the browser renders this

        
    
          
  1. Browser parses HTML top-to-bottom.
    2. 
          
  2. Hits <toggle-switch>. The tag isn't registered yet — that's fine. It creates a generic HTMLElement with that tag name and keeps going. CSS sees it as :not(:defined) and hides it.
    3. 
          
  3. Reaches the <script>. Runs customElements.define('toggle-switch', ToggleSwitch).
    4. 
          
  4. Browser walks the document, finds every existing <toggle-switch> instance, and upgrades it: swaps its prototype to ToggleSwitch, then fires connectedCallback. :defined now matches and the element appears.
    5. 
          
  5. Any <toggle-switch> created later (e.g. from document.createElement) is upgraded immediately on insertion.
    6. 
        


        
No build step required

        

          Web components are a native browser API. customElements, shadow DOM,
          <template>, ES classes, private fields, ElementInternals — all of it ships in
          modern browsers. You write a .html file, you double-click it, it works.
        


        
When you'd want imports / multiple files

        

          Once you have more than a handful of components, inline scripts get unwieldy. Switch to ES modules:
        


// toggle-switch.js
export class ToggleSwitch extends HTMLElement { /* ... */ }
customElements.define('toggle-switch', ToggleSwitch);

// rating-stars.js
export class RatingStars extends HTMLElement { /* ... */ }
customElements.define('rating-stars', RatingStars);

// app.js
import './toggle-switch.js';  // side-effect import: registers the tag
import './rating-stars.js';
// ...page wiring here


<!-- index.html -->
<script type="module" src="./app.js"></script>

        

          Catch: ES module imports require an HTTP server (any will do — python3 -m http.server, npx serve, GitHub Pages). They won't work over file:// because of CORS. Inline <script> still works fine from a local file.
        


        
When a build step starts to earn its keep

        
    
          
  • You want to use TypeScript.
    • 
          
  • You want to write template syntax (Lit's html`...`) without losing editor support.
    • 
          
  • You want to bundle dozens of modules into one file for production HTTP/1 latency. (HTTP/2 makes this less important.)
    • 
          
  • You want CSS preprocessing or hashed asset names for cache busting.
    • 
        

        

          For everything else — and certainly for learning — skip it.
